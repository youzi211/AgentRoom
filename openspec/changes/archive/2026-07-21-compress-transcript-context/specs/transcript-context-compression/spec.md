## ADDED Requirements

### Requirement: 单条历史消息在拼入提示词时受字符长度上限约束
系统在将房间历史消息拼装进 Agent 提示词的 transcript 段落时，SHALL 对每条消息正文施加确定性的字符长度上限；当消息正文超出该上限时，系统 SHALL 截断该正文并附加明确的省略标记，使 Agent 能够识别该内容并非完整原文。

#### Scenario: 单条消息未超长
- **WHEN** 一条历史消息的正文长度不超过单条字符上限
- **THEN** 该消息正文原样拼入 transcript 段落，不做任何截断或标记

#### Scenario: 单条消息超出长度上限
- **WHEN** 一条历史消息的正文长度超过单条字符上限
- **THEN** 系统截断该正文至上限长度
- **THEN** 截断后的文本附加明确的省略标记，说明该消息内容已被截断

### Requirement: 历史消息按新旧分层呈现
系统 SHALL 将拼入提示词的历史消息窗口划分为"全文层"和"摘要行层"：窗口内较新的消息属于全文层，按现有格式呈现（受单条字符长度上限约束）；较早的消息属于摘要行层，SHALL 以更简短的单行形式呈现，仅保留发言人、发言人类型和内容前缀，不保留完整正文。

#### Scenario: 窗口内较新消息保持全文呈现
- **WHEN** 一条历史消息位于当前滑动窗口的全文层范围内
- **THEN** 该消息按发言人、发言人类型和（必要时截断的）正文格式完整呈现

#### Scenario: 窗口内较早消息降级为摘要行
- **WHEN** 一条历史消息位于当前滑动窗口的摘要行层范围内
- **THEN** 该消息呈现为仅含发言人、发言人类型和内容前缀的单行文本
- **THEN** 该单行文本不包含消息的完整正文

### Requirement: transcript 总体拼装体积受字符预算约束
系统 SHALL 为拼入提示词的 transcript 段落设定一个总体字符预算；当分层呈现后的 transcript 总长度仍超出该预算时，系统 SHALL 从最旧的消息开始进一步压缩或丢弃条目，直至总长度满足预算或达到最小保留条数下限。

#### Scenario: 分层呈现后总体未超预算
- **WHEN** 经过单条截断和新旧分层处理后的 transcript 总字符数未超过总体预算
- **THEN** 系统不再对 transcript 做进一步压缩或丢弃

#### Scenario: 分层呈现后总体仍超预算
- **WHEN** 经过单条截断和新旧分层处理后的 transcript 总字符数超过总体预算
- **THEN** 系统从最旧的消息开始进一步压缩其呈现内容或将其整条移除
- **THEN** 系统持续该压缩过程直至总字符数满足预算，或已达到预先定义的最小保留消息条数

#### Scenario: 已达最小保留条数仍超预算
- **WHEN** transcript 已压缩至预先定义的最小保留消息条数，但总字符数仍超过总体预算
- **THEN** 系统停止进一步丢弃消息条目，保留该最小条数对应的内容
