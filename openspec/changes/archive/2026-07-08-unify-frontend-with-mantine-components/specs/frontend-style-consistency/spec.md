## ADDED Requirements

### Requirement: 主要页面使用 Mantine 控件体系
前端主要用户可见页面 MUST 优先使用现有 Mantine 组件呈现通用 UI 控件，包括页面容器、按钮、表单输入、文本域、下拉框、徽标、头像、弹窗、列表和数据表格。

#### Scenario: 页面控件统一到 Mantine
- **WHEN** 用户访问入口页、会议室页、管理后台、直接加入页、只读页或错误页
- **THEN** 页面中的通用按钮、表单控件、徽标和容器由 Mantine 组件承担，而不是各页面分别维护外观不同的原生控件样式

#### Scenario: 不新增 UI 组件库
- **WHEN** 实现前端风格统一
- **THEN** 系统 MUST 继续使用已有 `@mantine/core` 和 `@mantine/hooks` 依赖，不引入新的第三方 UI 组件库

### Requirement: 页面风格保持一致
前端页面 MUST 保持一致的浅色工作台风格、绿色强调色、紧凑信息密度、圆角尺度、按钮尺寸和状态表达，使入口、会议室、管理后台和只读页面看起来属于同一产品。

#### Scenario: 会议室与入口页视觉一致
- **WHEN** 用户从入口页进入实时会议室
- **THEN** 会议室顶部栏、侧栏、主要内容区和操作按钮 MUST 与入口页保持一致的色彩、边框、阴影和控件尺寸语言

#### Scenario: 管理后台与用户页面视觉一致
- **WHEN** 管理员进入会议管理或 Agent 管理页面
- **THEN** 管理后台 MUST 使用与用户页面一致的页面背景、导航、面板、表单和操作按钮风格，同时保持后台信息展示的扫描效率

### Requirement: 业务行为保持不变
前端风格统一 MUST NOT 改变现有路由、API 调用、WebSocket 连接、房间生命周期、管理员鉴权、Agent 提及、消息展示、会议纪要生成/编辑/下载或只读访问限制。

#### Scenario: 房间工作流不变
- **WHEN** 用户创建、加入、离开会议室，发送消息，插入 Agent 提及，或查看会议消息
- **THEN** 系统 MUST 保持与改造前一致的业务行为和数据交互

#### Scenario: 管理工作流不变
- **WHEN** 管理员校验密钥、管理会议、归档房间、编辑 Agent 或管理纪要版本
- **THEN** 系统 MUST 保持与改造前一致的表单提交、禁用态、错误提示和成功反馈

### Requirement: 自定义 CSS 仅保留应用特定布局
前端 CSS MUST 聚焦业务布局、响应式约束和特殊交互，已由 Mantine 替代的通用按钮、输入框、面板、徽标和弹窗样式 MUST 被移除、收窄或停止新增。

#### Scenario: 业务布局样式被保留
- **WHEN** 页面需要三栏工作台、可拖拽宽度、消息气泡、自动补全弹层、历史消息列表或只读页面网格
- **THEN** 系统 MAY 使用项目自定义 CSS 保持这些应用特定布局和交互

#### Scenario: 通用控件样式不再扩散
- **WHEN** 新增或改造页面中的通用按钮、表单、徽标、面板或弹窗
- **THEN** 系统 MUST 优先使用 Mantine 组件和组件属性，不继续复制新的全局自定义控件类
